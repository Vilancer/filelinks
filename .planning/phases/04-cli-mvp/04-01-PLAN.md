  <action>
Change **`loadFileLinksConfig`** to accept a **second argument** `options?: { readonly configPath?: string }` (or equivalent TypeScript shape). Behavior:

1. If **`options?.configPath`** is **absent** or **undefined**: preserve **current** behavior — call **`findConfigFile(startDir)`**; if `null`, throw **`ConfigNotFoundError`** as today; else load that discovered file.
2. If **`options.configPath`** is **present**: **resolve** it with **`path.resolve(startDir, options.configPath)`** when relative, or use as-is when already absolute (use **`path.isAbsolute`** + **`path.resolve`** consistently). **Do not** call **`findConfigFile`** for discovery. If the resolved path **does not exist** on disk, throw **`ConfigNotFoundError`** with a message that includes the resolved path.
3. Factor internal loading so both branches call a **single** private helper like **`loadFileLinksConfigFromFile(file: string): LoadedFileLinksConfig`** that contains the existing **jiti** + default export validation + **`FileLinksFileSchema`** decode logic currently after finding `file`.

Update **`export type`** / JSDoc on **`loadFileLinksConfig`** so callers see the new signature. Keep **`findConfigFile`** exported unchanged.
</action>
